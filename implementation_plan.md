# Baby Events Tracker - Implementation Plan

## Overview

A Progressive Web App (PWA) for tracking baby events (feeds, naps, potty) with a calendar view, daily timeline, and data persistence using IndexedDB.

---

## Tech Stack

- **Framework**: React 18 with Vite
- **Styling**: CSS (no external UI libraries)
- **Storage**: IndexedDB via idb library
- **PWA**: Vite PWA plugin
- **Date Handling**: date-fns
- **Icons**: Lucide React
- **Notifications**: React Toastify

## Data Models

### Event Types

```javascript
const EVENT_TYPES = {
  FEED: 'feed',
  NAP: 'nap',
  POTTY: 'potty'
};
```

### IndexedDB Schema

- **Store Name**: `events`
- **Key**: `id` (auto-generated)
- **Indexes**:
  - `by-date` (keyPath: `date`)
  - `by-date-time` (keyPath: `dateTime`)

### Event Objects

#### Base Event

```javascript
{
  id: string,           // UUID
  type: 'feed' | 'nap' | 'potty',
  date: string,         // YYYY-MM-DD
  timeFrom: string,     // HH:mm in 24hr format
  notes: string,        // Optional notes
  createdAt: string,    // ISO timestamp
  updatedAt: string     // ISO timestamp
}
```

#### Feed Event

```javascript
{
  ...baseEvent,
  type: 'feed',
  foodItems: [
    {
      type: 'milk',
      amount: number    // 0-300 ml
    },
    {
      type: 'solids',
      name: string,
      amount: number    // 10-300 mg
    }
  ]
}
```

#### Nap Event

```javascript
{
  ...baseEvent,
  type: 'nap',
  timeTo: string        // HH:mm in 24hr format (required for naps)
}
```

#### Potty Event

```javascript
{
  ...baseEvent,
  type: 'potty',
  isPee: boolean,
  isPoop: boolean
}
```

## Component Architecture

### App.jsx (Root)

- State: currentView ('calendar' | 'timeline')
- State: selectedDate
- State: events (all events)
- Navigation: Header with tabs for Calendar/Timeline
- Renders: CalendarView or DayTimeline based on currentView
- Includes: ToastContainer for notifications

### CalendarView.jsx

- Month grid display
- Navigation: Previous/Next month
- Events: Shows event markers on days with events
- Click: Select a date to view timeline
- Days without events: Clickable to create new event

### DayTimeline.jsx

- State: selectedDate
- Displays: All events for the selected date sorted by timeFrom
- EventItem: Individual event display with edit/delete options
- Empty state: Message when no events for day

### EventForm.jsx (Modal)

- Props: eventType, onSubmit, onCancel, initialData (for edit)
- Renders: Specific form based on eventType (FeedForm, NapForm, PottyForm)
- Validation: Date and timeFrom required for all events

### FeedForm.jsx

- State: foodItems array
- Add Food Item: Button to add milk or solids
- Milk Item:
  - Dropdown to select "milk"
  - Slider: 0-300 ml
  - Display: Current amount
- Solids Item:
  - Input: Name of food
  - Slider: 10-300 mg
  - Display: Current amount
- Remove: Button to remove each food item

### NapForm.jsx

- TimePicker for timeFrom
- TimePicker for timeTo
- Validation: timeTo must be after timeFrom (optional enforcement)

### PottyForm.jsx

- Switch component for Pee (on/off)
- Switch component for Poop (on/off)
- Validation: At least one must be selected

- Service worker for offline functionality

- Download button in header

## Key Technical Decisions

### Why IndexedDB?

- Larger storage than localStorage
- Asynchronous operations
- Indexable for efficient queries
- Supported in all modern browsers

### Why date-fns over Moment.js?

- Smaller bundle size
- Modular (tree-shakable)
- Immutable by default
- Function-based API

### Why Lucide React for icons?

- Lightweight and tree-shakable
- Consistent design language
- Wide variety of icons
- Easy to use with React

### Why React Toastify for notifications?

- Simple API
- Customizable positions and animations
- Auto-close functionality
- Success/error/info variants built-in

### Why CSS over Tailwind?

- No build step complexity
- Full control over styles
- Smaller runtime footprint
- As per user requirements

## Validation Rules

### All Events

- Date required
- timeFrom required
- timeFrom must be valid time format (HH:mm)

### Feed Events

- At least one food item
- Milk amount: 0-300 ml
- Solids amount: 10-300 mg
- Solids name required when type is solids

### Nap Events

- timeTo required
- timeTo should be after timeFrom (enforced in UI)

### Potty Events

- At least one: isPee or isPoop must be true

## User Flow

### Creating an Event

1. User opens app (displays calendar or timeline)
2. User clicks "+" button or selects a date
3. Modal opens with EventForm
4. User selects event type
5. User fills type-specific fields
6. User clicks Save
7. Event saved to IndexedDB
8. Success toast notification shown
9. Calendar/Timeline updates

### Editing an Event

1. User views event in timeline
2. User clicks edit icon (Lucide Pencil icon)
3. Modal opens with populated EventForm
4. User modifies fields
5. User clicks Save
6. Event updated in IndexedDB
7. Success toast notification shown
8. Timeline updates

### Deleting an Event

1. User views event in timeline
2. User clicks delete icon (Lucide Trash icon)
3. Confirmation toast or direct deletion
4. Event removed from IndexedDB
5. Success toast notification shown
6. Timeline updates

### Exporting Data

1. User clicks Export button (Lucide Download icon)
2. All events fetched from IndexedDB
3. Event data transformed to CSV format
4. File download triggered
5. Info toast notification shown
